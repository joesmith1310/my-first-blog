Project 1 - Java game

{% load static %}
{% load static %}
{% load static %}

My first project was a game created using only java and javafx. The game is called 'Pumba's Munch' inspired by the cartoon 'The Lion King'. The game involves moving the selected character in order to collect 'grubs' that give points, whilst dodging rocks which deduct lives.

The key things I learned during this project were how to use javafx to create an nice GUI, how to recieve input from a user's keyboard and mouse and move elements in the javafx scene in response. I also learned how to organise application files and link resources into my code to enable interesting graphics and interactive sounds. As well as this, I improved my understanding of java objects and the relationship between the components i.e. (pane, scene, stage) of a java fx application.

See how this game works

The application works by first creating a ViewManager object which has methods to create RunSubscene objects, as well as methods which create RunButton, InfoLabel and TextBox objects which make up the GUI on the main menu page. Each of these objects use classes I created which describe the width, height, colour etc. and include methods containing event handlers which respond to mouse events. The RunSubscene extends javafx Subscene and displays different sections of the GUI, it has a method which allows the subscene to be moved in and out of view using transition. The start button creates a GameViewManager object when clicked and executes the 'createNewGame' function which is passed the selected character, hides the menu stage and displays the game stage, which triggers the second part of the application if you like.

The GameViewManager object has a constructor which initializes the new game stage and creates 'keylisteners' which respond to the left and right arrow keys. the 'CreateNewGame' method, which is also called in response to the mouse click on the start button, calls numerous other methods including: 'createGameElements' which creates InfoLabel objects for the score and new ImageViews for the player lives, rocks and bugs. Crucially, 'createNewGame' also calls the 'createGameLoop' method. 'createGameLoop' uses an AnimationTimer to repeatedly call different methods for each frame so the game elements move. In every frame, 'moveCharacter' is called, a method which monitors the boolean 'leftKeyPressed' and 'rightKeyPressed', and then moves the player chracter node accordingly along the x-axis using transition. An interesting method is 'checkIfElementsCollide' which uses pythagoras' theorem to determine if the distance from one node to another is less than the radius of the two nodes added together and therefore whether or not they collide. Lives can then be deducted or points added based on collisions between nodes. My game monitors the y-position of the falling grubs and rocks and simply repositions them above the window when they fall below view in the 'checkElementsBelowView' to give the illusion of infinite falling nodes. There is also a basic levelling up system which simply counts how many rocks pass the player, and then increase the difficulty by adding more rock nodes to the scene. If you're using Firefox or Chrome on desktop you will be able to watch a video below which shows the application being used...

In terms of video games this application is VERY basic, however it did greatly improve my knowledge of javafx, as well as introducing me to new ways of interacting with the user which I hadn't explored in my course studies before.

My second project which I did at the start of summer, was a social media app programmed in swift using Xcode storyboard. The app is aimed at Birmingham university students and allows users to meet new people by matching them based on their profile. For example, a students profile may say that they live in the Vale (Student village at UOB), that they do computer science and they are from the West Midlands. This person would match well with someone who also lived on the Vale, studied another EPS subject like Engineering and was from the East Midlands. However, neither of these people would have a high percentage match with a student living in Selly Oak (The other side of campus) who studies History and is from the North East. Obviously this isn't to say that these people can't be friends, but the idea is that users can meet other users who they have things in common with.

The app is called UNIte and is easy for students to sign up and use. Users must give a profile picture, username, email and password, however this will not allow users to experience the full functionality of the app. To make the app useful, users must provide information about themselves so they can be matched with people more effectively. Users can provide information such as their real name and a bio, however these will not influence how users are matched. The basic factors that affect users compatability are the area where they're from, their university year, the department in which they study and the accomodation that they live in. The app is divided into to three main pages: 'My account', the feed and 'Friends'. 'My account allows users to view and edit their own information, the feed is a scrollable list of other users displaying their percentage match and the friends page allows users to send and recive friend requests as well as see their list of friends. Users can also visit another user's page, which displays the other user's information as well as a button to add/ remove as a friend. There is also a 'reveal' button which when pressed will reveal the user's preferred contact information. This app is not intended for messaging, there are plenty of ways to do that, more to find new people to message.

Technically, this project was more challenging than my first. My app uses a Google firebase database to store and access data so creating this app gave me some experience working with a database, structuring the data directories efficiently and writing simple security rules to protect user's data. My database has a hierachical tree structure where the highest level contains the user ID of each user, and then each user ID node has 4 branches: 'Friends', 'Recieved requests', 'Sent requests' and 'User data'. The first 3 just contain other user's user ID, and the 'User data' contains everything specific to that users profile e.g name, username, location etc.

To briefly summarise, when a user edits their account a function called 'updateUserCode' is called. This function generates a code string for that user depending on the information they have entered, for example it might append '56.' for one value, '20.' for another and then 'xx.' where the user has left a field blank (The numbers are closer together depending on how similar the categories they represent are). This results in a code such as '20.20.xx.27.' which is now stored in the 'User data' branch of the database. The 'FeedViewController' creates a 'CodeComparator' obejct and passes it the user's code as well as the code for one other user, it does this for every user in the feed. 'CodeComparator' works by taking a string (the code) and splitting it where there is a '.' and then converting the spliced string into integers. These integers represent information from the users account, for example 'west midlands' may be '45' and then 'east midlands' may be '50'. The difference between the numbers for each category is calculated and then averaged out across the categories, this number is then used to determine how similar these two profiles are.

During this project I have gained experience with another programming language, Swift, which has introduced me to ideas such as delegation and protocols which I had not come across before, although my knowledge of Java and general object-orientated programming helped me transfer over to Swift relativley easily. I know how to create database snapshots to access data and handle errors thrown e.g when requested data isn't available. Also, I can recieve user input from various methods e.g drop-down boxes and use their input to influence my programs. As a result of this project I am now proficient at designing and creating effective and user-friendly mobile GUIs. I implemented the use of segues and gestures to create intuitive user experience and I now have better understanding of what makes apps and programs in general easy to use.

This section showcases one of the websites I have worked on: www.knightonmanorcareservices.co.uk. This website has been custon built from scratch and is specifically tailored to the needs of the local business. Effective communication with the client was key in producing this website, it was important that all the information the clients wanted their customers to see was on the site and that it was easy to navigate to this information. The client entrusted me with a large creative influence on the project. I listened carefully to the image the company wanted to present, in this case they wanted to modernise their business so they could better compete with other companies. I designed the navigation system, as well as the website layout and theme to try and reflect these needs, as well as provide an exceptional user experience to the end-user. My online course in web design taught me colour theory, typography and how to layout a wep page effectively and I was able to use these skills to convey the business image the client desired.
